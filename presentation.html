Key Design Decision: Movie/TV data is NOT stored in our database. We only store tmdb_id as a reference and fetch details in real-time from TMDb. This keeps our DB lightweight and data always current.

API Data Fetching Strategy

Our backend acts as a proxy layer between the frontend and TMDb API.

1
React calls our Django API
→
2
Django's TMDbService makes server-side request
→
3
TMDb responds with JSON data
→
4
Django forwards response to frontend

Why Proxy Through Django?

  • API Key Security — Key stays on server, never exposed to browser
  • Data Shaping — Combine TMDb data with our own (reviews, favorites)
  • Rate Limit Control — Single point for caching and throttling

Search Functionality

🎬

Movie Search

Search movies by title with optional year filter

/api/search/movies/?query=Inception&year=2010
📺

TV Search

Search TV shows by title with pagination

/api/search/tv/?query=Breaking+Bad
🔍

Multi Search

Search across movies AND TV shows at once

/api/search/multi/?query=Marvel

Advanced Browsing — Discover Endpoint

Filter by genre, release year, and sort order (popularity, rating, date)

/api/discover/?genre=28&year=2024&sort_by=popularity.desc

Pagination: All results return 20 items per page with page parameter support for infinite scroll.
